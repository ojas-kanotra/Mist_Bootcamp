CSS


CSS (Cascading Style Sheets) controls presentation of HTML: colors, typography, spacing, layout and visual effects; use CSS to separate design from content so HTML stays semantic and accessible; where to put CSS: inline (style="..." on elements, quick, not recommended), internal ( inside an HTML file, useful for small pages or demos), external (, best practice); basic syntax: selector { property: value; } e.g. p.intro { color: #333; font-size: 16px; margin-bottom: 1rem; }; key concepts: cascade (later rules can override earlier ones), specificity (inline > ID > class > element), inheritance (some properties inherit), !important (forces override, avoid using it); box model: content, padding, border, margin; use box-sizing: border-box; layout tools: Flexbox and Grid for modern layouts, float/position for older patterns (use sparingly), media queries for responsive design @media (max-width: 768px) { ... }; modern features & workflow: CSS variables (--main-color: #06f;), preprocessors (Sass, Less), post-processing (autoprefixer, minification, bundlers), use DevTools to inspect styles and debug; accessibility & performance: keep CSS small and specific, avoid unused rules, ensure sufficient color contrast and keyboard-focus styles, load critical CSS early and defer large stylesheets when appropriate; best practices: prefer external stylesheets and utility/class-based approach, use meaningful class names and keep selector specificity low, organize styles by component or feature and document patterns.
